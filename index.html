Drug Prescribing for Dentistry: A technical decision

Agenda

Key topics discussed in this presentation:

  • background
  • technical decision
  • outcomes

Background

The Scottish Dental Clinical Effectiveness Programme (SDCEP):

  • Provides clinical guidance for dentists
  • Drug Prescribing for dentistry
  • Accurate, up to date, accessible

Needs

  • Rebuild existing app
  • Cost effective solution
  • Easy to update and manage
  • Accessible
  • Locked down to the UK

Constraints

tight deadline

UK lockdown

App experience

Options

  • Native app
  • Website
  • Progressive Web App (PWA)

Native app

Pros:

  • More functionality
  • Better performance
  • More security options
  • Offline access
  • Native app experience

Cons:

  • Less reach
  • Build time and cost
  • Harder to distribute
  • Difficult to update

Website

Pros:

  • More reach to users
  • Trackable on Google Analytics
  • Easy content management with Umbraco
  • Information always up to date
  • Quick build
  • Low cost

Cons:

  • No offline access
  • Lacking native app experience

Progressive Web App

Pros:

  • All the positives of a website
  • Easier to install than native app
  • Easy content management with Umbraco
  • Offline access
  • Can provide a native app-like experience

Cons:

  • Not allowed in Apple store
  • Harder for Safari users to bookmark

Decision

Progressive Web App

  • Responsive website
  • Native app-like experience
  • App user interface
  • App user interface
  • App user interface

Risks

Risks associated with the decision

  • Harder for Apple users to get app-like experience
  • Not geo locking to UK

const isInStandaloneMode = () =>
('standalone' in window.navigator)

const toMatch = [

/webOS/i,
/iPhone/i,
/iPad/i,
/iPod/i,
/BlackBerry/i,
/Windows Phone/i
];

return toMatch.some((toMatchItem) => navigator.userAgent.match(toMatchItem));
};

countryRestriction.js

// Block countries
function block_countries() {
const blocked_countries = [
"DE", // Germany
"US", // United States
"GB", // United Kingdom
"UA", // Ukraine
"AR", // Argentina
"FI", // Finland
"CA", // Canada
"JP", // Japan
"NO", // Norway
"RU" // Russia
]

// Detecting the users country
function get_country_code(api_url) {
fetch(api_url, { method: 'GET' })
// Getting ip info as json
.then(response => response.json())
.then(result => {
// If my ip country code is in blocked list
if (blocked_countries.includes(result.country)) {
display_access_denied_error() // Access denied error
}
})
.catch(error => console.log('error', error))
}

// Getting country code from third party api
get_country_code("https://get.geojs.io/v1/ip/country.json")
}

block_countries() // Block blocked countries

countryRestriction.js

// Allow countries
function allow_countries() {
const allowed_countries = [
"GB", // United Kingdom
]

// Detecting the users country
function get_country_code(api_url) {
fetch(api_url, { method: 'GET' })
// Getting ip info as json
.then(response => response.json())
.then(result => {
// If my ip country code is not in allowed list
if (!allowed_countries.includes(result.country)) {
display_access_denied_error() // Access denied error
}
})
.catch(error => console.log('error', error))
}

// Getting country code from third party api
get_country_code("https://get.geojs.io/v1/ip/country.json")
}

allow_countries() // Allow allowed countries

How to build a Progressive Web App

  • Favicons for different devices
  • JSON manifest file
  • Service worker

Service worker

Network first approach

  • User visits your website and is prompted to install the PWA
  • User accepts, JavaScript Service Worker is installed on their device.
  • Subsequent GET requests to your website are routed (proxied) via the Service Worker
  • Service Worker provides previously retrieved content from local device cache where appropriate or if user is offline.
